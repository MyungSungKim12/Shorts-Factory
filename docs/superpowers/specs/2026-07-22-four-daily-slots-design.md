# 하루 4회 운영 설계

## 목표

기존 11:00·17:00·21:00 KST 업로드를 유지하면서 14:00 KST 미스터리 영상을 추가한다.

## 설계

- 내부 회차 번호는 기존 데이터 호환을 위해 유지한다: slot 1=11시, slot 2=17시, slot 3=21시.
- 새 일일 두 번째 업로드는 이미 정의된 미스터리 카테고리인 slot 4를 사용해 14시에 실행한다.
- 14시 영상은 평상시 12시에 사전 제작하고, 정시에는 검증된 결과를 재사용해 업로드한다.
- 오늘처럼 사전 제작 시각이 지난 경우 `run_scheduled.py 4`를 즉시 실행해 제작부터 업로드까지 완료한다.
- 전역 잠금, 품질 게이트, 일 6건 제한, 7일 보관 정책은 그대로 유지한다.

## 운영 일정

- 09:00 slot 1 사전 제작 → 11:00 업로드
- 12:00 slot 4 사전 제작 → 14:00 업로드
- 15:00 slot 2 사전 제작 → 17:00 업로드
- 19:00 slot 3 사전 제작 → 21:00 업로드
- 06:30 검증 캐시 워밍은 유지한다.

## 검증과 복구

- 14시 선택·명시적 slot 4 선택·CLI 허용을 자동 테스트한다.
- 서버 코드·환경·자격 증명·데이터·cron을 배포 전에 백업한다.
- 원격 전체 테스트와 컴파일, 대시보드 상태, cron 9개를 확인한다.
- 즉시 실행은 업로드 ID, SQLite 레코드, 실행 로그, Telegram 알림으로 확인한다.
