# 스토리형 Shorts 프레임 레이아웃·제목 인트로 설계

## 목표

스토리형 Shorts가 본론부터 갑자기 시작하지 않도록 주제를 먼저 읽는 인트로를 추가한다. 영상 전체에 상·하단 검은 영역을 유지하고, 상단에는 제목을 고정하며 모든 자막은 하단 영역에 표시한다. 한국어 단어가 줄 중간에서 분리되는 자막 오류를 제거한다.

## 적용 범위

- 앞으로 제작되는 모든 `CONTENT_FORMAT=story` 영상에 적용한다.
- 교체 대상 `20260720-2`, YouTube 영상 `UgWHv40LnYQ`에도 동일하게 적용한다.
- 기존 순위형 렌더러는 변경하지 않는다.
- 기존 YouTube 영상 삭제는 사용자가 직접 수행한다.

## 화면 구성

최종 캔버스는 1080×1920을 유지한다.

- 상단 검은 영역: `y=0..259`, 높이 260px
- 중앙 영상 영역: `y=260..1589`, 높이 1330px
- 하단 검은 영역: `y=1590..1919`, 높이 330px

모든 이미지와 영상은 중앙 1080×1330 영역에 맞춰 확대·크롭한다. 가로형 Wikimedia 이미지는 전체 구도를 보존하는 전경과 흐린 배경 구성을 중앙 영역 안에서 유지한다. 상단과 하단 영역은 모든 장면, 인트로, CTA에서 검은색으로 고정한다.

## 상단 제목

- `script.json.title`을 전체 영상 동안 상단 검은 영역 중앙에 표시한다.
- 흰색 굵은 글씨와 약한 그림자를 사용한다.
- 1줄을 우선하고, 길면 공백·문장부호 기준으로 최대 2줄까지 나눈다.
- 단어 내부를 나누지 않는다.
- 2줄에도 맞지 않으면 글자를 자르지 않고 글자 크기를 단계적으로 줄인다.
- 제목은 장면 전환과 관계없이 같은 위치에 고정한다.

## 제목 인트로

1. 본문 TTS 전에 제목 전용 Neural2-C 음성을 별도로 생성한다.
2. 제목 문구는 `script.json.title`을 그대로 사용한다.
3. 첫 번째 본문 장면에서 선택된 첫 미디어를 인트로 영상에도 재사용한다.
4. 인트로 동안 첫 미디어는 정지 카드가 아니라 정상적으로 움직인다.
5. 제목 음성의 실제 길이와 0.15초 여백을 합친 길이만큼 인트로 클립을 만든다.
6. 인트로가 끝나면 현재 첫 훅 장면이 시작된다.
7. 인트로 제목 음성도 하단 검은 영역에 자막으로 표시한다.

## 자막 규칙

- 모든 자막은 하단 검은 영역 안에 가운데 정렬한다.
- 최대 2줄을 사용한다.
- 줄바꿈은 공백 또는 문장부호 경계에서만 허용한다.
- `300일 동안`을 `30` / `0일 동안`처럼 단어 내부에서 분할하지 않는다.
- 공백이 없는 긴 토큰은 자르지 않고 한 줄에 유지하며, 필요한 경우 해당 자막의 글자 크기를 줄인다.
- SRT 자막 덩어리 분할도 같은 토큰 보존 규칙을 사용한다.
- 자막의 시작·종료 시점은 기존처럼 실제 TTS 음성 길이를 사용한다.
- CTA 자막에도 같은 규칙을 적용한다.

## 길이 관리

- 최종 길이는 제목 인트로, 본문, CTA를 모두 포함해 60~75초여야 한다.
- 작가의 새 본문 목표는 53~58초로 조정해 제목 음성과 CTA 여유를 확보한다.
- 기존 대본을 교체 렌더링할 때는 실제 인트로·본문·CTA 음성의 합이 75초 이내인지 먼저 검사한다.
- 75초를 넘으면 음성 속도를 임의로 높이지 않고 제작을 중단한다.

## 제작 로그

`produce_log.json`에 다음 정보를 추가한다.

- `intro.text`
- `intro.audio_duration`
- `intro.padding_duration`
- `intro.tts.provider`, `voice`, `speaking_rate`
- `layout.top_band`, `video_height`, `bottom_band`
- 최종 제목 줄 수

## 교체 영상 절차

1. 서버의 기존 `topic.json`, `script.json`과 출처 정보를 재사용한다.
2. 새 레이아웃과 인트로를 적용해 별도 교체 파일로 렌더링한다.
3. 1080×1920, H.264/AAC, 60~75초, 오디오 존재, 검은 프레임 비율을 검사한다.
4. 시작·본문·CTA 프레임을 추출해 제목 위치와 하단 자막을 육안 확인한다.
5. 교체 영상 위치를 사용자에게 제공한다.
6. 사용자가 기존 YouTube 영상을 삭제했다고 확인한 뒤 DB의 기존 행 상태를 `replaced`로 변경한다.
7. 같은 회차의 새 영상만 공개 업로드하고 새 `video_id`를 DB에 기록한다.
8. 새 영상 URL과 DB 기록을 교차 검증한다.

## 오류 처리

- 첫 미디어를 구하지 못하면 기존 폴백 이미지를 인트로에도 사용한다.
- 제목 TTS가 Google에서 실패하면 기존 TTS 폴백 정책을 적용하고 실제 공급자를 로그에 기록한다.
- 인트로를 포함한 총 길이가 범위를 벗어나면 업로드 전에 실패한다.
- 제목 또는 자막이 영역을 벗어나면 렌더링 테스트와 프레임 검사에서 실패 처리한다.
- 사용자의 기존 영상 삭제 확인 전에는 교체본을 공개 업로드하지 않는다.

## 테스트

- 제목을 먼저 합성하고 본문보다 앞에 배치하는지 테스트한다.
- 인트로 길이가 제목 MP3 실제 길이와 0.15초 여백의 합인지 테스트한다.
- 최종 길이가 75초를 넘을 때 실패하는지 테스트한다.
- 상단 260px·중앙 1330px·하단 330px 필터 구성을 테스트한다.
- 제목이 최대 2줄이고 단어 내부를 나누지 않는지 테스트한다.
- `300일 동안`과 공백 없는 긴 토큰이 중간에서 잘리지 않는지 테스트한다.
- 자막 스타일이 하단 검은 영역 안에 배치되는지 테스트한다.
- 기존 순위형 테스트와 전체 회귀 테스트를 실행한다.
