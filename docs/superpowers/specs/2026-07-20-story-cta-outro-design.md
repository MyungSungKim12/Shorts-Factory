# 스토리형 Shorts CTA 엔딩 설계

## 목표

앞으로 제작되는 스토리형 Shorts의 본문 뒤에 주제 맞춤형 구독·좋아요 요청을 음성과 자막으로 자동 추가한다. 기존 순위형 렌더러와 이미 생성된 샘플은 변경하지 않는다.

## 범위

- `CONTENT_FORMAT=story`로 새로 제작되는 영상에만 적용한다.
- `script.json`의 `cta`를 엔딩 문구의 원본으로 사용한다.
- 기존 순위형 영상과 렌더러에는 적용하지 않는다.
- 업로드 및 스케줄 설정은 변경하지 않는다.

## CTA 문구 규칙

1. 작가가 영상 주제와 자연스럽게 연결되는 `cta`를 작성한다.
2. 문구에 `구독`과 `좋아요`가 모두 포함되어야 한다.
3. 둘 중 하나라도 빠지거나 `cta`가 비어 있으면 다음 고정 문구로 보완한다.
   - `이런 이야기가 더 궁금하다면, 구독과 좋아요 부탁드립니다.`
4. CTA는 한두 문장, 약 3~4초 분량으로 제한한다.

## 제작 흐름

1. 본문 장면의 Neural2 음성과 영상을 기존 방식으로 제작한다.
2. 정규화된 CTA 문구를 동일한 Neural2-C 설정으로 별도 합성한다.
3. CTA 음성의 실제 재생 시간을 측정한다.
4. 마지막 본문 장면의 시각 소스를 재사용해 CTA 길이만큼 독립 엔딩 클립을 만든다.
5. 엔딩 화면은 가벼운 암전 오버레이를 적용해 자막 가독성을 높인다.
6. CTA 자막은 실제 CTA 음성 길이에 맞춰 표시하고 음성이 끝나면 함께 종료한다.
7. 본문과 CTA 엔딩을 합성한 뒤 최종 영상을 검증한다.

## 길이 관리

- 최종 결과는 CTA를 포함해 60~75초여야 한다.
- 작가의 본문 목표 길이를 기존 60~65초에서 57~62초로 조정해 CTA 여유를 확보한다.
- 실제 Neural2 음성과 CTA를 합친 예상 길이가 75초를 넘으면 제작을 중단하고 대본 축약이 필요하다는 오류를 반환한다. 음성 속도를 임의로 높여 제한을 우회하지 않는다.

## 실패 처리

- 주제 맞춤형 CTA가 비었거나 필수 표현이 빠지면 고정 문구를 사용한다.
- CTA TTS는 본문과 동일하게 Google Neural2를 우선 사용하며 기존 TTS 폴백 정책을 따른다.
- CTA 미디어는 마지막 본문 소스를 재사용하므로 추가 미디어 API 실패 지점을 만들지 않는다.
- CTA를 포함한 최종 영상이 60~75초, 1080×1920, H.264/AAC 조건을 충족하지 않으면 샘플 검증을 실패 처리한다.

## 기록

`produce_log.json`에 다음 정보를 추가한다.

- 최종 CTA 문구
- CTA 음성 길이
- CTA에 사용한 TTS 공급자·보이스·속도
- CTA 자동 보완 여부
- CTA를 포함한 최종 영상 길이

## 테스트

- 비어 있거나 필수 표현이 빠진 CTA가 고정 문구로 보완되는지 검사한다.
- 올바른 주제 맞춤형 CTA가 그대로 유지되는지 검사한다.
- CTA 자막 종료 시점이 실제 CTA 음성 길이와 일치하는지 검사한다.
- CTA를 포함한 예상 길이가 75초를 넘으면 제작이 중단되는지 검사한다.
- 순위형 렌더러가 변경되지 않았는지 전체 회귀 테스트로 확인한다.
- `story-v3` 샘플을 렌더링해 해상도, 코덱, 오디오, 검은 화면 비율과 전체 길이를 자동 검증한다.

## 샘플 및 원복

- 새 샘플은 `data/samples/story-v3/`에 생성한다.
- `story-v1`, `story-v2`는 비교와 원복을 위해 보존한다.
- 기능은 스토리형 제작 경로에만 추가되므로 기존 순위형으로 원복할 때 별도 데이터 변환이 필요하지 않다.
