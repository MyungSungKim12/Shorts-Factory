# 클릭을 부르는 스토리 소재 선정 설계

## 목표

추가 API 호출 없이 현재 리서처 호출 한 번으로 무료 스톡 제작이 가능하면서도 사람들이 즉시 궁금해할 소재를 선택한다.

## 선정 방식

리서처는 최종 JSON을 만들기 전에 내부적으로 후보 8개를 비교한다. 각 후보를 다음 여섯 항목에서 0~5점으로 평가한다.

- 첫 3초 호기심
- 상식 반전
- 위험·규모·충격
- 남성 시청자 관심도
- 무료 스톡 확보 가능성
- 최근 소재와의 차별성

총점 24점 이상을 목표로 한다. 모든 후보가 목표에 못 미쳐도 회차를 중단하거나 재호출하지 않고 최고점 후보를 반드시 선택한다. 후보 목록과 평가 과정은 출력하지 않고 최종 소재 JSON에 `interest_score`와 `selection_reason`만 기록한다.

## 회차별 방향

- slot 1: 극한 생존과 위험한 동물
- slot 4: 미스터리와 기이한 기록
- slot 2: 금지된 장소와 거대 구조
- slot 3: 역사적 반전·재난·치명적 실수

단순 색깔의 이유, 이름 유래, 평범한 형성 과정, 교과서형 설명은 제외한다. 무료 영상 확보성은 필수 조건이지만 재미보다 먼저 소재를 결정하지 않는다.

## 할당량과 레거시 정리

- 별도 심사 에이전트나 추가 모델 호출을 만들지 않는다.
- 기존 소재 호출 횟수와 06:30 캐시 워머를 포함한 운영 cron은 변경하지 않는다.
- 실행되지 않는 `agents/*.md`의 랭킹 전용 설명을 현재 스토리 역할에 맞춘다.
- 분석 문서에서 실제로 연결되지 않은 자동 학습 루프 표현을 제거한다.
- 랭킹 호환 코드 전체 삭제는 이번 범위에 포함하지 않는다.

## 검증

사용자 요청에 따라 샘플 영상과 전체 회귀 테스트는 생략한다. 기존 소재 프롬프트·회차 관련 테스트와 Python 컴파일만 실행한다.
