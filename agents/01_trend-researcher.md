# 소재 리서처 (Trend Researcher) — 스토리 포맷

## 역할

무료 영상으로 표현할 수 있으면서 사람들이 제목만 보고도 결말을 궁금해할 단일 소재를 고른다. API 호출을 늘리지 않기 위해 한 번의 모델 호출 안에서 후보 8개를 비교하고 최고점 소재 하나만 출력한다.

## 입출력 계약

- 입력: 회차별 스토리 방향, 최근 14일 소재, 검색 그라운딩 사용 가능 여부
- 출력: `data/work/{run_id}/topic.json`
- 실제 계약 구현: `app/models.py`의 `StoryTopicContract`
- 실제 실행 프롬프트: `app/agents/researcher.py`의 `_story_researcher_prompt()`

## 소재 선정 기준

후보마다 다음 항목을 0~5점으로 평가한다.

1. 첫 3초 호기심
2. 상식 반전
3. 위험·규모·충격
4. 남성 시청자 관심도
5. 무료 스톡 확보 가능성
6. 최근 소재와의 차별성

24/30 이상을 목표로 한다. 목표 점수에 못 미쳐도 재호출하거나 회차를 중단하지 않고 최고점 후보를 선택한다.

## 회차 방향

- slot 1: 극한 생존과 위험한 동물
- slot 4: 미스터리와 기이한 기록
- slot 2: 금지된 장소와 거대 구조
- slot 3: 역사적 반전·재난·치명적 실수

## 제외 대상

- 단순 색깔의 이유, 이름 유래, 평범한 형성 과정
- 답이 바로 예상되는 교과서형 질문
- 구체적인 위험·반전·규모 없이 분위기만 신비로운 소재
- 무료 스톡이 많다는 이유만으로 선택한 풍경·귀여운 동물 소개
- 최근 14일 소재와 사실상 같은 이야기

## 검증 경로

- 검색 성공: `grounded_search`
- 검색 실패 후 기존 검증 소재 사용: `verified_cache`
- 둘 다 불가능한 경우: 변하지 않는 사실만 쓰는 `model_memory`

검증 방식과 출처는 항상 `topic.json`에 기록한다.
